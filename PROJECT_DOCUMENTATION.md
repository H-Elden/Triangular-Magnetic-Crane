## 1. 项目概述

### 1.1 作品简介

本产品为一台**无轨道三角式双臂磁吸起重机**。三棱柱形碳纤维框架稳定性高、重量轻。起重机配有三个可动双电磁铁取物装置，辅以引导机构增大容许误差，通过反转正负极快速消除剩磁；闭环步进电机驱动齿轮齿条起升，同步带传动实现水平移动。行进模块采用GMR编码器控制的直流减速电机，单轴陀螺仪矫正直线行进误差。采用机器前端的超声波测距传感器实现提前感知和平稳停车。

### 1.2 创新点

- **碳纤维框架**: 三棱柱形碳纤维框架，轻量高强，提升整机负载能力和运行效率
- **提前识别与减速**: 超声波传感器提前感知目标物，主动减速实现平稳停车
- **电磁铁吸取与消磁电路**: 双电磁铁取物装置 + 自设计消磁电路，继电器断电后通入反向电流消除剩磁

### 1.3 应用前景

起重机能高速运行并一次性吸取三个物品，稳定高效，定位精确，小型轻量，降低了能耗。同时适用于：
- **传统物流场景**: 仓库、生产线、分拣中心
- **轻量化场景**: 随车起重、海上作业平台等有较高轻量化要求的物流场景

### 1.4 技术指标

| 指标 | 参数 |
|------|------|
| 主控芯片 | STM32F103ZET6, 72MHz |
| 驱动方式 | 双轮差速驱动 |
| 电机类型 | 行星减速电机 (27:1) + GMR编码器 (500 P/R, 轮趣科技) |
| 电机驱动 | D50A大功率MOS双路直流有刷电机驱动模块 (12A/24V, 轮趣科技) |
| 控制频率 | 100Hz (10ms周期) |
| 传感器 | 3×HC-SR04超声波, 1×JY901S九轴陀螺仪 |
| 抓取机构 | 5×闭环步进电机, 3×双电磁铁 |
| 最大速度 | 约 0.89 m/s (MVEL=2500 时) |
| 框架材料 | 碳纤维三棱柱 |

## 4. 系统初始化流程

`main()` 调用 `Init()` 完成以下初始化序列：

```
1.  TIM5_Init()                  → 100ms 系统节拍 (用于步进电机计时)
2.  NVIC_PriorityGroupConfig()   → 中断优先级分组 (Group2)
3.  delay_init()                 → SysTick 延时函数
4.  LED_Init()                   → 红绿LED (PB5, PE5)
5.  KEY_GPIO_Init()              → 按键KEY0 (PE4)
6.  Usart2Init(9600)             → 陀螺仪通信口
7.  Usart3_Init()                → 步进电机通信口 (115200)
8.  WitInit() + 回调注册         → WIT陀螺仪SDK初始化
9.  AutoScanSensor()             → 自动扫描陀螺仪波特率
10. WitSetOutputRate()           → 设置10Hz数据输出
11. WitSetUartBaud(115200)       → 切换至115200通信
12. Gyro_read() ×2              → 读取初始Yaw角作为航向基准 (ZhongZhi)
13. PointDis_Init()              → 初始化点位距离表
14. TIM6_Init()                  → 100Hz主控制定时器
15. TIM7_Init()                  → 1us分辨率超声波计时器
16. MotorEncoder_Init_TIM2/4()   → 左右编码器初始化
17. Motor_Init()                 → 电机驱动方向引脚初始化 (D50A)
18. LL298N_Init() / RL298N_Init()→ PWM输出初始化 (10kHz), 文件名沿用L298N为历史原因
19. HCSR04_Init()               → 超声波传感器 + EXTI中断
20. Switch_Init()               → 6路继电器初始化
21. 发送步进电机查询命令          → 验证通信连接 (红灯检测)
22. LED_GREEN = 0               → 绿灯亮，初始化完成
```

## 5. 运动控制系统

### 5.1 底盘驱动系统

#### 5.1.1 硬件参数

| 参数 | 值 |
|------|-----|
| 电机类型 | 行星减速电机 (轮趣科技) |
| 减速比 | 27:1 |
| 编码器 | GMR巨磁阻编码器, 500 P/R, AB相正交 |
| 编码器分辨率 | 27 × 500 × 4 = **54,000 Counts/Rev** |
| 车轮半径 | 32.0 mm |
| 每脉冲行进距离 | 2×π×32 / 54000 ≈ **0.00372 mm** |
| PWM频率 | 10 kHz (TIM3/TIM8, ARR=7199, PSC=0) |
| 控制频率 | 100 Hz (TIM6, 10ms周期) |

#### 5.1.2 驱动接口 (D50A大功率MOS驱动)

> **注意**: 代码文件名沿用 `HARDWARE/L298N.c`, `HARDWARE/L298N.h`，这是因为复用之前L298N模块代码时保留了文件名。实际使用的硬件是轮趣科技的 **D50A大功率MOS双路直流有刷电机驱动模块**（12A大电流，支持24V）。

**文件**: `HARDWARE/L298N.c`, `HARDWARE/L298N.h`

| 信号 | 左轮 | 右轮 |
|------|------|------|
| 方向IN1 | PB12 | PB14 |
| 方向IN2 | PB13 | PB15 |
| PWM | PB1 (TIM3_CH4) | PC9 (TIM8_CH4) |

`SetPWM(motorLeft, motorRight)`:
- 正值 → 正转 (IN1=0, IN2=1)
- 负值 → 反转 (IN1=1, IN2=0)
- PWM占空比 = CCR值 / 7200

#### 5.1.3 速度与距离换算

根据实验标定经验公式：
$$
实际速度 v(m/s) = 3.57 × 10^{-4} × \text{PWM值}
$$
MVEL=2500 时，理论最大速度约 **0.89 m/s**。

### 5.2 编码器读数

**文件**: `HARDWARE/MotorEncoder.c`

```c
// 左编码器 (TIM4)
int LRead_Encoder(void) {
    Encoder_TIM = TIM4->CNT;
    if (Encoder_TIM > 0xEFFF) Encoder_TIM -= 0xFFFF;  // 符号转换
    TIM4->CNT = 0;  // 清零
    return Encoder_TIM;
}
// 右编码器 (TIM2) 同理
```

编码器工作于**正交编码模式3** (TI12, 四倍频)，每10ms读取一次差值作为速度反馈。

**距离累积**:
```c
Run_Dis += -1.0 * LEncoder / 54000 * 2 * PI * Radius;  // mm
```

### 5.3 级联PID控制器

系统采用**级联PID**架构，在`VelCir`状态（精确停车）下同时运行：

```
目标位置 ──→ [位置PID] ──→ [速度限幅] ──→ [速度PI] ──→ [PWM输出] ──→ 电机
  ↑                                                              │
  └──────────────────── 编码器反馈 ────────────────────────────────┘
```

#### 速度PI控制器（增量式）

```c
ControlVelocity += Kp × (Bias - Last_Bias) + Ki × Bias
Kp = 1.0, Ki = 0.5
```

#### 位置PID控制器（位置式）

```c
ControlVelocity = Kp × Bias + Ki × ΣBias + Kd × (Bias - Last_Bias)
Kp = 0.25, Ki = 0.02, Kd = 1.0
```

积分限幅: [-970, 970]，防止积分饱和。

#### 速度限幅

位置PID输出限制在 `[-2×TargetVelocity, +2×TargetVelocity]`，然后减半后送入速度PI控制器。

### 5.4 加减速控制

```c
// 加速阶段: 目标速度从 StartMinV 线性增加到 TargetVelocity
LTargetVelocity = Accel × CurrentPosition / 54000 + StartMinV

// 减速阶段: 目标速度线性降到 StartMinV
LTargetVelocity = Velocity_temp - Dccel × CurrentPosition / 54000
```

| 参数 | 值 |
|------|-----|
| Accel (加速率) | 1200 |
| Dccel (减速率) | 800 (首次1400) |
| StartMinV | ±400 |

加速判断逻辑: 当累计编码器脉冲超过目标圈数的一半时，从加速切换到减速。

### 5.5 陀螺仪航向修正

在`Velocity_Xunji`模式下，陀螺仪PD修正叠加到左右轮目标速度：

```c
Turn = Kp × (YAW - ZhongZhi) + Kd × (Bias - Last_Bias)
Kp = 500, Kd = 2000, 输出限幅 [-50, 50]
```

修正值**同相叠加**到左右轮（左轮减Turn，右轮减Turn），通过差速产生旋转力纠正航向。

#### 预设角度

```c

S2 = 2065°  // 水平摆动大角度

C2 = 960°   // 垂直大行程
```

#### 时序控制

步进电机采用**开环时序预估**方式判断运动完成（非闭环反馈）:

- TIM5产生100ms系统时基
- `Stepper_Turn()` 根据角度和是否带载计算预计完成时刻 `close_time[addr]`
- `Stepper_GetStatus(addr)` 通过比较当前时间与预计完成时刻判断是否到位
- 带载时（`weight[addr-3]=1`），预计时间延长:
  - TIME_C1_W=8 vs TIME_C1=4
  - TIME_C2_W=15 vs TIME_C2=5
  - TIME_Z0_W=10 vs TIME_Z0=4

## 6. 传感器系统

### 6.1 超声波测距 (HC-SR04 ×3)

**文件**: `HARDWARE/HC-SR04.c`, `HARDWARE/HC-SR04.h`

#### 引脚分配

| 传感器 | 位置 | Echo | Trig |
|--------|------|------|------|
| 0 | 前方中间 | PG0 | PG1 |
| 1 | 前方左侧 | PE7 | PE8 |
| 2 | 前方右侧 | PE9 | PE10 |

#### 工作原理

1. **触发**: `Getdis()` 向已开启的传感器发送20μs的Trig高电平脉冲
2. **计时**: Echo引脚上升沿触发EXTI中断，记录TIM7计数值; 下降沿再次触发，计算时间差
3. **转换**: `dist = 0.0001 × cnt × 340 / 2` (单位: cm)
4. **滤波**: `countEffect()` 滑动窗口滤波，15个采样周期内 ≥3次有效才确认

#### 滑动窗口滤波算法

```c
// ones[i]: 位掩码，每bit代表一次测量的有效性
// pos[i]: 当前写入位
// CYCLES = 15 (窗口大小)
ones[i] &= ~(1 << pos[i]);      // 清除旧位
if (input) ones[i] |= (1 << pos[i]);  // 写入新位
pos[i] = (pos[i] + 1) % CYCLES;
return __builtin_popcountll(ones[i]);  // 统计有效次数
```

#### 检测API

| 函数 | 说明 |
|------|------|
| `SensorON(i)` | 开启传感器i |
| `SensorOFF(i)` | 关闭传感器i |
| `SetGoaldis(i, min, max)` | 设置有效距离窗口 (cm) |
| `Getdis()` | 触发一次所有已开启传感器 |
| `countEffect(i, input)` | 滑动窗口滤波 |

### 6.2 陀螺仪姿态检测 (JY901S)

**文件**: `HARDWARE/gyroscope.c`, `HARDWARE/gyroscope.h`, `HARDWARE/wit_c_sdk.c`, `HARDWARE/wit_c_sdk.h`, `HARDWARE/REG.h`

#### 连接参数

| 参数 | 值 |
|------|-----|
| 接口 | USART2 (PA2 TX, PA3 RX) |
| 协议 | WIT标准串口协议 (0x55帧头) |
| 最终波特率 | 115200 |
| 输出速率 | 10Hz |
| 数据类型 | 加速度、角速度、角度、磁场 |

#### 初始化流程

1. `AutoScanSensor()` → 遍历4800~921600波特率自动识别传感器
2. `WitSetOutputRate(RRATE_10HZ)` → 设置10Hz回传
3. `WitSetUartBaud(WIT_BAUD_115200)` → 设置115200波特率
4. 读取两次Yaw角取均值作为航向基准`ZhongZhi`

#### 数据读取

```c
// 每次TIM6中断 (10ms) 中调用
fAcc[i]   = sReg[AX+i] / 32768.0 × 16.0      // 加速度 (g)
fGyro[i]  = sReg[GX+i] / 32768.0 × 2000.0    // 角速度 (deg/s)
fAngle[i] = sReg[Roll+i] / 32768.0 × 180.0    // 欧拉角 (deg)
```

#### 坐标轴定义

| 寄存器 | 含义 | 寄存器地址 |
|--------|------|-----------|
| AX/AY/AZ | 加速度 | 0x34-0x36 |
| GX/GY/GZ | 角速度 | 0x37-0x39 |
| Roll | 横滚角 | 0x3D |
| Pitch | 俯仰角 | 0x3E |
| Yaw | 航向角 | 0x3F |

> 实际只使用了Yaw角（fAngle[2]）进行直线行进航向修正。这是一个重要的设计变更：早期方案采用TSL1401线性CCD传感器，通过采集视野范围内128像素的信息，对循迹带进行二值化识别来矫正直线行进。后来改为陀螺仪航向校准方案，**既不需要在地面上粘贴循迹带，也不需要铺设和固定轨道**，大大降低了场地准备的工作量和对场地的依赖。

## 7. 电磁铁与消磁系统

**文件**: `HARDWARE/magnet.c`, `HARDWARE/magnet.h`

### 7.1 硬件连接

6路继电器均连接在GPIOD:

| 通道 | 类型 | GPIO | 功能 |
|------|------|------|------|
| 0 | 吸合 | PD3 | 电磁铁0通电吸合 |
| 1 | 吸合 | PD1 | 电磁铁1通电吸合 |
| 2 | 吸合 | PD2 | 电磁铁2通电吸合 |
| 3 | 消磁 | PD6 | 电磁铁0反向消磁 |
| 4 | 消磁 | PD4 | 电磁铁1反向消磁 |
| 5 | 消磁 | PD5 | 电磁铁2反向消磁 |

### 7.2 消磁电路原理

```c
// 吸合
MagnetON(i):  GPIO_SetBits(吸合引脚)

// 释放 (含消磁)
MagnetOFF(i):
  1. GPIO_ResetBits(吸合引脚)        // 断电
  2. GPIO_SetBits(消磁引脚)          // 通入反向电流
  3. delay_ms(10)                    // 持续10ms
  4. GPIO_ResetBits(消磁引脚)        // 断开反向电流
```

利用**反向电流脉冲**快速消除铁芯中的剩磁，防止断电后工件仍被吸附。这是自设计的特色电路。

### 7.3 抓取时序

```
电磁铁对准 ──→ MagnetON ──(50ms)──→ 步进电机提升 ──→ 到达 ──→ MagnetOFF (含消磁)
    ↑                                     ↑                    ↑
   位置到达                             吸合确认            释放完成
```

```
去程:
  A ──→ B检测 → C检测+抓取(左右两个) → D放置中间物体 → 去H放置
回程:
  H ──→ F检测/补抓 → E检测/补抓 → I放置
```

### 8.5 抓取函数 Catch()

```c
void Catch(char Line, u8 odd, u8 mid, u8 even) {
    // 1. 等待前序步进电机运动完成
    // 2. 垂直步进电机下降至抓取高度
    // 3. 等待下降完成
    // 4. 延迟50ms
    // 5. 电磁铁通电吸合 (MagnetON)
    // 6. 延迟50ms
    // 7. 设置带载标志 (weight[])
    // 8. 垂直步进电机上升
    // 9. 水平步进电机外摆
}
```

参数中 `odd/even` 对应左右臂（传感器1/2），`mid` 对应中心（传感器0）。

## 9. 通信协议

### 9.1 步进电机串口协议 (USART3)

**物理层**:
- 接口: USART3 (PB10 TX, PB11 RX)
- 波特率: 115200
- 数据位: 8, 停止位: 1, 无校验

**命令帧格式** (13字节):

| 字节 | 内容 | 说明 |
|------|------|------|
| 0 | addr | 电机地址 (1-5) |
| 1 | 0xFD | 功能码 |
| 2 | dir | 方向 (0=CCW, 1=CW) |
| 3-4 | vel | 速度, RPM (高字节在前) |
| 5 | acc | 加速度 (0-255) |
| 6-9 | clk | 脉冲数 (32位, 高字节在前) |
| 10 | raF | 相对/绝对标志 (false=相对) |
| 11 | snF | 同步标志 (false=不启用) |
| 12 | 0x6B | 校验和 |

**脉冲数计算**: `clk = angle / 1.8 × 16` (微步细分因子)

**接收**: 使用FIFO缓冲区 (128×uint16_t) + IDLE中断检测帧结束。

### 9.2 陀螺仪WIT协议 (USART2)

**帧格式**: `0x55` 帧头 + 数据 + 校验和

陀螺仪以10Hz频率主动回传角度、角速度、加速度、磁场数据包。WIT SDK (`wit_c_sdk.c`) 负责解析协议。

## 10. 控制算法详解

### 10.1 PID控制算法

#### 增量式速度PI

```
公式:  u(k) = u(k-1) + Kp[e(k) - e(k-1)] + Ki × e(k)
特性:  只输出增量，无积分饱和问题，适合速度环
参数:  Kp=1.0, Ki=0.5
```

#### 位置式位置PID

```
公式:  u(k) = Kp × e(k) + Ki × Σe(k) + Kd[e(k) - e(k-1)]
特性:  直接输出控制量，需要积分限幅防止饱和
参数:  Kp=0.25, Ki=0.02, Kd=1.0
限幅:  积分项限制在 [-970, 970]
```

#### 航向修正PD

```
公式:  Turn = Kp × (YAW - ZhongZhi) + Kd × (Bias - Last_Bias)
参数:  Kp=500, Kd=2000
限幅:  输出限制在 [-50, 50] PWM
```

### 10.2 梯形速度曲线

```
速度
  ▲
  │        ┌──────────────────┐
  │       ╱                    ╲
  │      ╱                      ╲
  │     ╱                        ╲
  │    ╱                          ╲
  │   ╱                            ╲
  │  ╱                              ╲
  └──┴────┴──────────────────────────┴───→ 时间/距离
     加速段      匀速段      减速段          (Run模式)
```

- 加速阶段: 目标速度按 `Accel=1200` 斜率线性增加
- 减速阶段: 目标速度按 `Dccel=800` 斜率线性降低
- 切换到减速的判据: `|当前位置| > |目标圈数|/2`

### 10.3 超声波数字滤波

15位滑动窗口 + 多数表决: 在最近的15次测量中，至少3次落在目标距离范围内才确认检测到目标。此设计有效滤除超声波传感器的偶发干扰和噪声。

## 14. 创新点与技术特色

### 14.1 碳纤维框架

三棱柱形碳纤维框架兼顾了结构强度与轻量化，使整机在保持刚性的同时降低自重，减小了驱动电机的负载，提高了运行速度和能效。

### 14.2 自设计消磁电路

- **原理**: 电磁铁断电后，继电器切换通入5-10ms反向电流脉冲，主动消除铁芯剩磁
- **实现**: 每个电磁铁配备2路继电器 (吸合 + 消磁)
- **效果**: 有效解决断电后工件被剩余磁力吸附无法释放的问题

### 14.3 陀螺仪航向校准

- 替代传统的TSL1401线性CCD循迹带巡线方案
- 原方案需要在地面粘贴黑色循迹带，CCD采集128像素线阵图像进行二值化识别；改为陀螺仪方案后**既不需要粘贴循迹带，也不需要铺设和固定轨道**，大大降低了场地准备的工作量
- 利用JY901S九轴陀螺仪的Yaw角进行PD航向修正
- 每10ms在速度控制循环中叠加修正量，保证长距离直线行进精度

### 14.4 超声波提前感知

- 3个HC-SR04超声波传感器覆盖前方左中右三个方向
- 15位滑动窗口滤波，3/15表决机制，抗干扰性强
- 检测到目标后可**提前减速**并精确停车，避免急停冲击

### 14.5 级联PID精确停车

- 外环位置PID + 内环速度PI的**级联控制架构**
- 位置环含积分限幅，速度环采用增量式PI
- 梯形速度曲线，加减速平滑
- 停车精度达到**亚毫米级** (每编码器脉冲 ≈ 0.0037mm)

### 14.6 双模式任务策略

- `way=0`: 优先在E/F/G点抓取
- `way=1`: 优先在C/Cy点抓取
- 根据B线是否检测到目标物自动切换策略，适应不同场地布局
